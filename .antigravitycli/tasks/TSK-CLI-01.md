# Задача: TSK-CLI-01 — Неинтерактивный CLI-интерфейс (argparse)

## 📌 Часть 1: Инструкция по выполнению (Implementation Guide)
1. **Цель**: Создать консольный интерфейс UDE для бесшовной интеграции и запуска компилятора в неинтерактивном режиме в CI/CD конвейерах.
2. **Шаги реализации**:
   * Создать файл `oda_ude/cli.py` в качестве главной точки входа.
   * Использовать библиотеку `argparse` для разбора флагов:
     * `--config`, `-c` (путь к файлу конфигурации проекта, например `ude_config.json`).
     * `--input`, `-i` (переопределение пути к исходному коду/XML, указанного в конфиге `src_dir`).
     * `--output`, `-o` (переопределение целевой папки для результатов, указанной в конфиге `output_dir`).
     * `--format`, `-f` (переопределение формата рендеринга: `hugo_markdown` или `html`).
   * Реализовать логику разбора:
     * Загрузка указанного `ude_config.json`.
     * Поиск родительского `product.json` в директории выше для обогащения метаданных каталога (название, версия продукта).
     * Разрешение относительных путей (`src_dir`, `output_dir`, `static_pages_dir`) относительно каталога, где расположен `ude_config.json` (`REQ-FUN-29`).
   * Обеспечить перехват любых исключений: вывод короткой понятной ошибки в `sys.stderr` и завершение с кодом **`1`** (или кодами 2-4 согласно SDD), а при успешном выполнении — завершение с кодом **`0`** (`REQ-FUN-07`).

## 🧪 Часть 2: Инструкция по проверке результата (Verification & TDD Scenarios)
1. **Тестовый сценарий (TDD Red Phase)**:
   * Написать `tests/test_cli.py`.
   * Использовать `pytest` для имитации консольного вызова (через monkeypatch или `sys.argv`).
   * Проверить:
     1. Запуск с валидными параметрами успешно отрабатывает и вызывает системный выход `SystemExit` с кодом `0`.
     2. Запуск с отсутствующими параметрами или неизвестным форматом вызывает выход с кодом `1` или `2`.
   * Тесты должны упасть.
2. **Реализация (TDD Green Phase)**:
   * Реализовать разбор аргументов в `cli.py` и связать парсинг и рендеринг в единую цепочку вызовов.
3. **Запуск и валидация (TDD Refactor Phase)**:
   * Запустить команду проверки:
     ```bash
     poetry run pytest tests/test_cli.py
     ```
   * **Ожидаемый успешный результат**: зеленый статус, консольный интерфейс полностью готов к автоматизации на серверах сборки.
