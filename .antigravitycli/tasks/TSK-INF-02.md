# Задача: TSK-INF-02 — Подготовка тестовых XML-ассетов Doxygen

## 📌 Часть 1: Инструкция по выполнению (Implementation Guide)
1. **Цель**: Подготовить тестовый стенд с неизменяемыми XML-ассетами Doxygen для тестирования парсинга и трассировки.
2. **Шаги реализации**:
   * Создать вспомогательный класс `MockAssetLoader` в `tests/utils.py`.
   * Создать директорию для хранения тестовых слепков: `tests/assets/doxygen/`.
   * Разместить в ней минимальные валидные XML-файлы, имитирующие реальную выгрузку Doxygen:
     * `index.xml` — корневой файл каталога сущностей.
     * `class_definition.xml` — файл класса (например, класс `MyClass` на C++ с публичными методами и полями).

## 🧪 Часть 2: Инструкция по проверке результата (Verification & TDD Scenarios)
1. **Тестовый сценарий (TDD Red Phase)**:
   * Написать тест `tests/test_assets.py`, который пытается инициализировать `MockAssetLoader().load_xml("index.xml")` и проверяет, что метод возвращает непустую строку (содержимое XML).
   * Убедиться, что тест падает, так как папки и вспомогательного класса не существует.
2. **Реализация (TDD Green Phase)**:
   * Создать класс `MockAssetLoader` в `tests/utils.py`, считывающий данные из `tests/assets/doxygen/`.
3. **Запуск и валидация (TDD Refactor Phase)**:
   * Запустить команду проверки:
     ```bash
     poetry run pytest tests/test_assets.py
     ```
   * **Ожидаемый успешный результат**: зеленый тест, подтверждающий корректный импорт и загрузку тестовых XML.
