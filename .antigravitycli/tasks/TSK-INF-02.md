# Задача: TSK-INF-02 — Подготовка тестовых XML-ассетов Doxygen

## 📌 Часть 1: Инструкция по выполнению (Implementation Guide)
1. **Цель**: Подготовить тестовый стенд с неизменяемыми XML-ассетами Doxygen и исходными файлами для тестирования парсинга, фильтрации макросов/SWIG и трассировки (соответствует Рекомендации 2 по развитию проекта).
2. **Шаги реализации**:
   * Создать вспомогательный класс `MockAssetLoader` в `tests/utils.py`.
   * Создать директорию для хранения исходных тестовых файлов (Mock-проектов): `tests/assets/src/`.
     * Разместить там минимальные синтетические файлы кода (например, `MyClass.h` с макросами экспорта `NWDBEXPORT` и `class_swig_wrapper.cs` с полями `swigCPtr` / методом `Dispose()`).
   * Создать директорию для хранения тестовых XML-слепков Doxygen: `tests/assets/doxygen/`.
   * Разместить в ней минимальные валидные XML-файлы, имитирующие реальную выгрузку Doxygen:
     * `index.xml` — корневой файл каталога сущностей.
     * `class_definition.xml` — файл стандартного C++ класса с публичными методами и полями.
     * `class_with_macros.xml` — файл класса с экспортными макросами (`NWDBEXPORT`, `MAPEXPORT` и т.д.) для проверки их автоматического отсечения (`REQ-FUN-19`).
     * `class_swig_wrapper.xml` — файл класса, содержащего служебную SWIG-обвязку (`swigCPtr`, `swigCMemOwn`, `Dispose()`), для проверки её фильтрации при соответствующем флаге (`REQ-FUN-20`).

## 🧪 Часть 2: Инструкция по проверке результата (Verification & TDD Scenarios)
1. **Тестовый сценарий (TDD Red Phase)**:
   * Написать тест `tests/test_assets.py`, который пытается инициализировать `MockAssetLoader().load_xml("index.xml")` и проверяет, что метод возвращает непустую строку (содержимое XML).
   * Убедиться, что также загружаются новые ассеты: `class_with_macros.xml` и `class_swig_wrapper.xml`.
   * Убедиться, что тест пападают, так как папки и вспомогательного класса не существует.
2. **Реализация (TDD Green Phase)**:
   * Создать класс `MockAssetLoader` в `tests/utils.py`, считывающий данные из `tests/assets/doxygen/`.
3. **Запуск и валидация (TDD Refactor Phase)**:
   * Запустить команду проверки:
     ```bash
     poetry run pytest tests/test_assets.py
     ```
   * **Ожидаемый успешный результат**: зеленый тест, подтверждающий корректный импорт и загрузку всех тестовых XML-ассетов.

## 👥 Часть 3: Инструкция по приемке пользователем (User Acceptance Scenario)
После завершения шагов 1 (разработка кода) и 2 (проверка тестами) со стороны ИИ, вам необходимо выполнить финальную приемку задачи:
