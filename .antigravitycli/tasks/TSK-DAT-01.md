# Задача: TSK-DAT-01 — Схемы валидации Pydantic v2 IR

## 📌 Часть 1: Инструкция по выполнению (Implementation Guide)
1. **Цель**: Создать типизированные Pydantic-схемы для промежуточного представления (IR) программного каталога API.
2. **Шаги реализации**:
   * Создать файл `oda_ude/models.py`.
   * Объявить Pydantic v2 модели, представляющие иерархию кода:
     * `ProjectCatalog` — корневой каталог (содержит список пространств имен).
     * `NamespaceEntity` — пространство имен, пакет или модуль.
     * `ClassEntity` — класс, интерфейс или структура (имя, docstring, методы, поля).
     * `MethodEntity` — функции, методы классов, конструкторы (имя, сигнатура, параметры, возвращаемый тип, нормализованный docstring).
     * `ParameterField` — параметр метода (имя, тип, описание).

## 🧪 Часть 2: Инструкция по проверке результата (Verification & TDD Scenarios)
1. **Тестовый сценарий (TDD Red Phase)**:
   * Написать `tests/test_models.py` с двумя проверками:
     1. Создание валидного дерева `ProjectCatalog` (вложенные классы, методы с аргументами) проходит успешно.
     2. Передача невалидного типа (например, число вместо строки в `fully_qualified_name`) вызывает валидационное исключение `pydantic.ValidationError`.
   * Убедиться, что тесты падают из-за отсутствия моделей.
2. **Реализация (TDD Green Phase)**:
   * Реализовать модели в `oda_ude/models.py`, унаследовав их от `pydantic.BaseModel` с точными аннотациями типов.
3. **Запуск и валидация (TDD Refactor Phase)**:
   * Запустить команду проверки:
     ```bash
     poetry run pytest tests/test_models.py
     ```
   * **Ожидаемый успешный результат**: тесты проходят успешно, типы и вложенные структуры строго валидируются.
