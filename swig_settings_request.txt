Уважаемый Администратор сборки SWIG-оберток!

Для корректной генерации и последующей компиляции технической документации API (с помощью Universal Documentation Engine — UDE) для Python-оберток, просим Вас настроить процесс генерации SWIG следующим образом:

1. В конфигурационные интерфейсные файлы (.i) для Python-оберток добавить директивы:
   
   %feature("autodoc", "3");
   
   Это обеспечит:
   - Генерацию детальных, PEP 484-совместимых сигнатур в docstrings Python (типы аргументов и тип возвращаемого значения, например: `def foo(x: int) -> float`).
   - Корректное распознавание прототипов функций встроенными инструментами статического анализа, IDE и парсером UDE.
   
   %feature("kwargs", 1);
   
   Это обеспечит:
   - Поддержку именованных аргументов (Keyword Arguments) в Python (например, `box.scale(factor=2.0)`).
   - Жесткую фиксацию реальных имен C++ параметров в сгенерированном Python-коде, предотвращая замену имен на абстрактные `arg1, arg2`.

2. В вызов утилиты swig на этапе компиляции добавить флаг:
   -doxygen
   
   Пример вызова:
   swig -python -doxygen -c++ <имя_файла>.i
   
   Это обеспечит:
   - Автоматическую трансляцию Doxygen-комментариев (Javadoc/Doxygen стилей: @param, @returns, @brief) из C++ заголовков в PEP 257-совместимые docstrings (в формате RST/Sphinx) внутри генерируемых Python-файлов оберток.
   - Возможность для движка UDE автоматически собирать детальные и структурированные таблицы параметров и методов в итоговой документации.

Без указанных настроек автоматическая генерация качественной документации API для Python-оберток будет невозможна.
